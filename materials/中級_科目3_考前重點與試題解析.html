3.1 機率/統計之機器學習基礎應用

核心定義機器學習的本質是學習條件機率分佈 P(Y|X)，X 為輸入特徵、Y 為目標變數；模型輸出機率而非確定答案，機率反映模型信心程度。

機率分佈分類（高頻考點）

類型描述函數常見分佈與用途
離散型機率質量函數 PMF；總和為 1伯努利（二元，邏輯迴歸基礎）、二項（n 次伯努利累計）、泊松（固定區間稀有事件、事件獨立、平均率為常數）
連續型機率密度函數 PDF；單點機率為 0，須積分求區間機率常態（誤差建模、標準化）、均勻（初始化/抽樣）、指數（等待時間、無記憶性）、卡方（標準常態平方和、統計檢定）
易考陷阱連續變數用 PDF（非 PMF）；伯努利取值僅 0 或 1；邏輯迴歸假設目標服從伯努利分佈，線性迴歸假設誤差服從常態分佈。

條件機率與貝氏定理

  • 條件機率：P(A|B) = P(A∩B) / P(B)
  • 貝氏定理：P(A|B) = P(B|A)·P(A) / P(B)
  • 四要素：先驗 P(A)、似然 P(B|A)、邊際 P(B)、後驗 P(A|B)（核心輸出）。樸素貝氏假設特徵條件獨立，故 P(X|C)=∏P(xᵢ|C)。

統計推論與假設檢定

項目參數估計假設檢定
目的推測母體參數的值/區間驗證關於母體參數的主張是否成立
輸出點估計 + 信賴區間（如 95% CI）p 值、檢定統計量，比較顯著水準後決策
  • 顯著水準 α：虛無假設為真時錯誤拒絕的容忍機率，即型一錯誤機率（常設 0.05）。
  • p 值：觀察樣本後算出的機率，衡量資料與虛無假設的相符程度；p < α 則拒絕虛無假設。
  • 統計量：期望值（平均趨勢）、變異數/標準差（離散程度）、偏態（對稱性，過大需 Log/Box-Cox）、峰度（尖峰厚尾）。中位數不受極端值影響。

3.2 線性代數之機器學習基礎應用

  • 向量運算：點積（相似度/投影，線性模型預測核心）、L2 範數（向量長度，正則化基礎）。
  • 矩陣運算：乘法（批次預測、特徵轉換）、轉置、求逆/偽逆（最小平方解）。神經網路前向傳播 z=Wa+b 本質為矩陣乘法。
  • 矩陣相乘維度：(1,10)×(10,64) → (1,64)；內維須相符，否則不可乘。

矩陣分解與降維

方法形式用途
特徵值分解 (EVD)A=QΛQᵀ（對稱方陣）PCA、LDA 找投影方向
奇異值分解 (SVD)X=UΣVᵀ（任意矩陣）降維、推薦系統、LSA、影像壓縮
非負矩陣分解 (NMF)X≈WH（皆非負）主題建模、可加疊部件、稀疏解構
PCA 記憶點對協方差矩陣做特徵值分解，保留特徵值最大的前 K 個方向（最大變異方向）。解釋變異比例 = 該主成分特徵值 ÷ 全部特徵值總和。線性、正交、對離群值敏感、可解釋性低。

3.3 數值優化技術與方法

訓練模型 = 最小化損失函數的優化問題。要素：目標函數、決策變數、可行域、函數性質（凸性/可導性）。凸問題（線性/邏輯迴歸）有全域最優保證；非凸問題（神經網路）存在多個局部最優解與鞍點。

損失函數對應任務

任務損失函數特性
迴歸MSE 均方誤差平方懲罰，放大大誤差/離群值梯度
MAE 平均絕對誤差線性懲罰，對離群值不敏感，梯度近固定
Huber 損失小誤差平方、大誤差線性，折衷抗雜訊
分類交叉熵 / Log Loss衡量機率分佈差距，分類標準選擇
Focal Loss強化難分類樣本，適合不平衡

梯度下降與優化器

  • GD（全批次）：用全部資料算梯度，穩定但慢、耗記憶體。
  • SGD（隨機）：每次一筆，快但震盪大、不穩定。
  • Mini-batch SGD：折衷，深度學習業界標準（批次如 256–2048），平衡梯度穩定性與 GPU 平行效率。
優化器機制備註
Momentum累積過去梯度產生慣性減少震盪、加速收斂、跳脫局部解
Adagrad累積歷史梯度平方，逐參數調學習率適合稀疏特徵；學習率會持續衰減至極小
RMSprop滑動平均修正 Adagrad 衰減適合 RNN/非穩定梯度
Adam結合 Momentum + RMSprop（一階+二階動量），含偏差修正萬用優化器，收斂快穩；記憶體較高
穩定訓練策略學習率調度（Step/Cosine/ReduceLROnPlateau）、早停 Early Stopping、梯度裁剪 Gradient Clipping（防梯度爆炸/NaN）、Batch Normalization。學習率過大 → 損失震盪或發散；過小 → 收斂極慢。

4.1 機器學習原理與技術

學習類型資料任務 / 演算法
監督式有標籤分類（離散）、迴歸（連續）
非監督式無標籤聚類、降維、關聯規則、表示學習
強化式環境互動回饋序列決策，最大化長期報酬；MDP（Agent/Environment/State/Action/Reward/Policy/Value）
  • 資料分割：訓練集（學參數）、驗證集（調超參、監控過擬合）、測試集（最終評估）。
  • 分類指標：Accuracy、Precision、Recall、F1、ROC-AUC；迴歸指標：MAE、MSE、R²。

4.2 常見機器學習演算法

監督式 — 迴歸

  • 線性迴歸 y=β₀+β₁x₁+…+βₙxₙ+ε；五大假設：線性、誤差常態、變異數齊一、誤差獨立、無多重共線性。對離群值敏感、無法捕捉非線性。
  • Lasso (L1)：絕對值懲罰 λΣ|β|，使部分係數歸零→特徵選擇；高度相關特徵中隨機保留其一（重要性不穩定）。
  • Ridge (L2)：平方懲罰 λΣβ²，縮小但不歸零，共線性下穩定估計、保留全部特徵。
  • SVR：ε-tube 內誤差忽略，可用核函數做非線性；對離群值有抵抗力。
  • 決策樹迴歸：葉節點取樣本平均；非參數、不需標準化、易過擬合、輸出分段不連續。

監督式 — 分類

演算法關鍵特性
邏輯迴歸Sigmoid 輸出機率；二元分類；假設 log-odds 線性、無共線性；高解釋性
SVM最大化間隔超平面；核函數處理非線性；高維小樣本佳；參數 C、γ
決策樹分裂依據：基尼不純度、資訊增益（熵）、分類誤差；遞迴分裂、易過擬合
KNN惰性學習、距離為基礎；須特徵縮放；維度詛咒；對 K 值敏感
樸素貝氏貝氏定理+特徵條件獨立；文本/垃圾信佳；零機率需拉普拉斯平滑

集成方法

策略機制代表
Bagging隨機子集獨立建模、投票；降變異Random Forest
Boosting序列修正前者殘差/錯誤；降偏差Gradient Boosting、XGBoost、LightGBM
XGBoost vs GBDTXGBoost 在目標函數加入正則化項（樹深、葉節點數、葉權重 L2），用二階導數(Hessian)加速、支援缺失值自動處理與並行化，採 Level-wise 生長；LightGBM 採 Leaf-wise（直方圖法），更快但小資料易過擬合。

非監督式

  • K-means：需指定 K、歐氏距離、僅球形/凸群、對初始化與離群值敏感、每次結果略不同；難處理非凸（半月形）。
  • 階層式：樹狀圖、無需指定 K、計算量大、合併不可逆。
  • DBSCAN：密度為基礎（Eps、MinPts）、任意形狀群、自動標記雜訊點；鄰域點數不足且不被任何核心點密度可達→歸為雜訊點。
  • 降維：PCA（線性、可作模型輸入）、t-SNE（非線性、保留局部鄰域、僅視覺化、不可外推、不適合作模型輸入）、UMAP（較快、保局部+全局）。
  • 關聯規則：Apriori（候選集+剪枝）、FP-Growth（FP 樹、兩次掃描）；支持度/信賴度/提升度(Lift>1 正相關)。
  • 異常偵測：Isolation Forest（孤立路徑短）、One-Class SVM（僅學正常邊界，僅需正常資料）。

4.3 深度學習原理與框架

  • 人工神經元：加權求和 z=Σwx+b → 激活函數。激活函數引入非線性，無激活則多層等同單層線性。感知器用步階函數、僅解線性可分（無法解 XOR）。
  • 前向傳播（做預測）→ 損失函數 → 反向傳播（連鎖律算梯度、調參數）。

5.1 數據準備與特徵工程

  • 缺失值：刪除、填補（均值/中位數/眾數、KNN、預測模型填補）、缺失指標編碼。
  • 異常值：Z-score、IQR（Q3+1.5×IQR）、Isolation Forest/LOF；處理為移除、截尾/轉換、或標記保留（如詐欺）。
  • 特徵選擇：Filter（統計量/相關係數、卡方）、Wrapper（RFE、前/後向）、Embedded（樹重要性、Lasso/Ridge，訓練中內建選擇）。
  • 尺度調整：Min-Max（0~1，對極端值敏感）、Z-score 標準化（平均 0 標準差 1）、Robust Scaling（中位數+IQR，抗離群）。
  • 分佈轉換：Log（右偏）、平方/立方根（中度偏態）、Box-Cox/Yeo-Johnson（自動近常態）。注意：對本已線性的特徵做 Log 反而破壞結構。

編碼適用風險

Target Encoding高基數且與目標相關須防資料洩漏
Embedding高基數（深度學習）—
  • 時間特徵：拆解年/月/星期/時段；週期性用 sin/cos 編碼。時序滑動窗口建立滯後特徵。
  • 資料增強：影像（翻轉/旋轉/裁切，注意水平翻轉會破壞數字/字母方向語義）、文字（同義替換）、時序（加噪）、表格（SMOTE）。

5.2 模型選擇與架構設計

  • 資料規模：小型→簡單模型（線性/邏輯/決策樹/正則化）；中型→RF/XGBoost/LightGBM；大型/非結構化→深度模型。
  • 解釋性：高（線性/邏輯/淺樹）；低（RF/GBT/DNN，搭配 SHAP/LIME/PDP）。SHAP提供單筆預測的局部可解釋性；Feature Importance 為全域。
  • 部署：資源受限（IoT/手機/邊緣）→簡單模型或剪枝/量化/蒸餾；即時推論重延遲。
偏差-變異權衡偏差(Bias)高=擬合不足=欠擬合（低複雜度模型）；變異(Variance)高=對雜訊敏感=過擬合（高複雜度模型）。訓練分高、驗證分低且不同切分差異大 → 高變異/過擬合。

5.4 模型調整與優化

  • 超參數（手動指定、非學習得出）：學習率、批次大小、網路深度/寬度、激活函數、優化器、正則化係數 λ。
  • 調參：Grid Search（窮舉，組合數=各參數候選相乘×fold 數）、Random Search（高維參數空間搜尋效率更高）、AutoML（缺工程師/時間時快速比較多模型）。

正則化與穩定化

技術作用
L1 (Lasso)絕對值懲罰、部分權重歸零、特徵選擇
L2 (Ridge / Weight Decay)平方懲罰、縮小權重但不歸零、共線性穩定
Elastic NetL1+L2，兼具稀疏與穩定
Dropout訓練時隨機關閉神經元、降依賴；測試時關閉
Early Stopping驗證不再改善即停（patience）；防過擬合
降複雜度 ≠ 增能力降低過擬合的策略是限制學習能力（L1/L2、Dropout、Early Stopping、減少特徵/參數）；「擴增輸入特徵」「增加 Epoch」反而提升能力，不屬降低過擬合。

模型壓縮與加速

  • 知識蒸餾：教師模型→學生模型，小而快、近似表現。
  • 剪枝 Pruning：移除低貢獻權重/神經元。
  • 量化 Quantization：32-bit 浮點→8-bit 整數，不需重訓即可縮小加速。
  • 混合精度訓練（FP16+FP32）：加速、省記憶體，需梯度縮放。

6 機器學習治理

數據隱私、安全與合規

  • 風險類型：直接識別（PII：姓名/身分證/Email）、間接識別（準識別：性別/生日/地理）、再識別（去識別後仍被還原）。
  • 基礎匿名化：遮蔽 Masking、雜湊 Hashing（單向不可逆）、泛化 Generalization、分桶 Bucketing、隨機擾動 Noise Injection（加隨機誤差）。
  • 進階 PETs：K-匿名（準識別欄位相同）、L-多樣性（敏感欄位多值）、T-接近性（分佈與整體相近，防分佈推測）、聯邦學習（本地訓練只傳參數）、同態加密（加密狀態下直接運算、不需解密）。
  • 合規：合法來源/告知同意、資料最小化與目的限制、去識別化、透明度與紀錄、跨境傳輸（GDPR/CCPA/PDPA）。

演算法偏見與公平性

  • 資料偏誤：來源偏誤（群體分佈不均/取樣偏差）、內容偏誤（刻板印象）、製程偏誤（標註主觀→標籤偏差）。
  • 公平性指標：Demographic Parity（各群正向比例相同）、Equal Opportunity（真正合格者被正確預測機率相同）、Equalized Odds（TPR+FPR 皆一致）、Disparate Impact（80% 法則）。
  • 去偏三階段：前處理（重抽樣、特徵去偏）、訓練中/模型內（公平性約束、對抗式去偏、公平性正規化）、後處理（門檻調整、結果校準）。
  • 治理：公平性審查委員會、多元團隊、定期稽核、資訊揭露；工具 AIF360、Fairlearn。

常見計算速查

  • F1=2PR/(P+R)。例：P=0.8、R=0.6 → 2·0.48/1.4 ≈ 0.686。
  • 條件機率 P(A|B)=A_and_B / B（分母為條件事件 B 的計數）。
  • PCA 解釋變異=該特徵值/Σ特徵值。例 λ=6,3,1（總和 10）→前兩主成分=90%。
  • Grid Search 模型數=各超參數候選值連乘 × fold 數。例 3×4×5×3×3×5=2700。
  • 矩陣相乘=(a,b)×(b,c)→(a,c)。例 (1,10)×(10,64)→(1,64)。
  • 時間複雜度：兩兩比對全部樣本 → O(n²)。
  • R²=1−RSS/TSS；R²=0.85 表示 85% 變異被解釋。

1.某零售企業建立一個銷售預測模型，希望評估該模型在不同月份的新資料上，是否仍能維持穩定的預測表現。資料科學團隊計畫利用統計方法檢驗模型對未觀察資料的適應能力與泛化效果。下列哪一種方法最適合用於此目的？

  • (A) F 檢定（F-test）
  • (B) 交叉驗證（Cross-Validation）
  • (C) 配對樣本 t 檢定（Paired-sample t-test）
  • (D) 卡方檢定（Chi-square Test）
答案：B
解析交叉驗證透過多次切分資料、輪流訓練與驗證，直接衡量模型在未見資料上的泛化能力與穩定性，正是評估「對未觀察資料適應能力」的標準工具。(A)(C)(D) 為假設檢定，用於檢驗統計差異而非評估模型泛化。

2.在建立迴歸或分類模型時，若希望避免模型過度擬合（Overfitting），可透過加入正則化項以限制模型的複雜度。其中，L1 正則化（Lasso）的主要效果為何？

  • (A) 增加模型參數的數量，以提升表現靈活度
  • (B) 強化梯度穩定性，避免參數更新過度震盪
  • (C) 產生稀疏模型（Sparse Model），使部分參數權重收斂為零
  • (D) 提高學習率（Learning Rate），加速模型收斂速度
答案：C
解析L1 正則化（Lasso）以權重絕對值和為懲罰項，能將不重要的係數縮為 0，產生稀疏模型並達成特徵選擇。L2(Ridge) 則僅縮小不歸零。

3.在訓練非線性模型時，若目標函數為非凸函數（Non-convex Function），演算法在參數更新過程中可能出現多個極值點，導致最佳化結果不穩定。請問此時最可能發生下列哪一種情況？

  • (A) 梯度消失
  • (B) 資料過少
  • (C) 局部最優解
  • (D) 過擬合
答案：C
解析非凸函數存在多個極值點與鞍點，梯度下降可能收斂到非全域的局部最優解，使結果不穩定。神經網路損失函數即屬此類。

4.在執行 DBSCAN 群集分析時，若某資料點鄰域內的樣本數不足以形成核心點（Core Point），且該點未被任何核心點的鄰域所包含，也未與其他群集形成密度可達關係（Density Reachability），此資料點最終將被歸類為哪一種類型？

  • (A) 鄰近點（Neighbor Point）
  • (B) 雜訊點（Noise Point）
  • (C) 邊界點（Border Point）
  • (D) 潛在點（Potential Point）
答案：B
解析DBSCAN 中，既非核心點、又不在任何核心點鄰域內（非邊界點）、亦無密度可達關係的點，會被標記為雜訊點/離群值。這正是 DBSCAN 能自動分離噪聲的特性。

5.某智慧製造公司開發影像辨識系統檢測瑕疵，採用 CNN 為主架構，其中第一層卷積層（Convolutional Layer）主要負責的功能為下列何者？

解析卷積層的核心功能是透過卷積核滑動，自動學習並提取輸入的局部特徵（如邊緣、紋理）。(B) 為池化層；(D) 為全連接/輸出層。

6.交通監控團隊發現 CNN 在訓練與推論效率上明顯優於傳統全連接神經網路（FCNN）。請問下列何者為主要原因？

  • (A) CNN 能自動學習影像的旋轉與比例不變性
  • (B) CNN 可直接跳過人工特徵提取步驟進行分類

7.下列哪一種應用最適合採用長短期記憶網路（LSTM）模型？

49.（AOI 題組）若需建立超過 50 層的深層神經網路捕捉細微劃傷特徵，又擔心傳統深層網路常見的梯度消失問題，下列哪一種經典架構最適合優先考慮？

  • (A) VGG：透過重複堆疊 3×3 卷積層與池化層來維持梯度穩定
  • (B) GoogLeNet：透過 Inception 模塊並行使用不同大小的卷積核
  • (C) ResNet（Residual Network）：透過殘差連接（Residual Connection / Skip Connection），使梯度能跨層傳遞並降低深層訓練困難

50.（AOI 題組）為修復訓練中 Loss 變 NaN，考慮在 PyTorch 訓練迴圈加入梯度裁剪（torch.nn.utils.clip_grad_norm_）。若要正確加入，最適合插入於哪個位置？其主要作用為何？

  • (A) 插入於位置 3 與位置 4 之間；用以限制 Loss 數值大小，避免梯度消失
